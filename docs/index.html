PCA

Principal Component Analysis (PCA) is used to:

- Reduce dimensionality

- Explore structure in data

- Identify patterns (e.g. batch effects, groups)

What PCA is

  • PCA converts many correlated variables into a smaller number of new variables called principal components.
  • PC1 explains the largest amount of variation in the dataset.
  • PC2 explains the next largest amount of variation, while being independent of PC1.
  • This helps us visualise high-dimensional data in two dimensions.

How to interpret a PCA plot

  • Each point represents one sample.
  • The axes show the sample scores on PC1 and PC2.
  • Points close together are more similar; points far apart are more different.
  • Separation by colour can suggest structure such as species, batch effects, or case/control grouping.
  • The scree plot helps decide how much of the total variation the first few components explain.

Perform PCA using prcomp

# PCA needs only numeric columns, so we subset those
pca_input <- iris[, c("Sepal.Length", "Sepal.Width", "Petal.Length", "Petal.Width")]

pca_result <- prcomp(pca_input, scale. = TRUE)

variance_explained <- summary(pca_result)$importance[2, ] * 100
pc1_label <- sprintf("PC1 (%.1f%%)", variance_explained[1])
pc2_label <- sprintf("PC2 (%.1f%%)", variance_explained[2])

summary(pca_result)

Visualize the PCA

Load visualisation packages

library(ggplot2)
library(patchwork)

Set up a presentation theme

pca_plot_theme <- theme_classic(base_size = 16) +
  theme(
    plot.title = element_text(face = "bold", size = 18, hjust = 0.5),
    axis.title = element_text(face = "bold", size = 14),
    axis.text = element_text(size = 12, colour = "black"),
    legend.title = element_text(face = "bold"),
    legend.text = element_text(size = 11),
    legend.position = "right",
    panel.border = element_rect(colour = "black", fill = NA, linewidth = 0.6),
    plot.background = element_rect(fill = "white", colour = NA),
    panel.background = element_rect(fill = "white", colour = NA),
    axis.line = element_blank()
  )

Add metadata back to the PCA results

pca_df <- as.data.frame(pca_result$x)

# Add metadata back
pca_df$Species <- iris$Species
pca_df$Group <- iris$Group
pca_df$Batch <- iris$Batch
pca_df$Age <- iris$Age

Important

Metadata must match the same samples used in the PCA. If these don’t match after filtering or NA handling, the colours will break.

Example: Effect of missing values on sample alignment

# Introduce some missing values
pca_input_na <- pca_input

# Remove rows with NA

Note

If rows are removed from the PCA input, the metadata must be filtered in the same way before colouring the PCA plot.

Create a PCA plot: code

ggplot(pca_df, aes(PC1, PC2, colour = Species)) +
  geom_point(size = 3, alpha = 0.85) +
  labs(
    title = "PCA plot coloured by Species",
    x = pc1_label,
    y = pc2_label,
    colour = "Species"
  ) +
  pca_plot_theme

Scree plot

Prepare data for visualisation

scree_pca_input <- data.frame(
  Component = 1:length(pca_result$sdev),
  Variance = pca_result$sdev^2 / sum(pca_result$sdev^2)
)

scree_pca_input
  Component    Variance
1         1 0.729624454
2         2 0.228507618

Create the scree plot: code

ggplot(scree_pca_input, aes(x = Component, y = Variance)) +
  geom_bar(stat = "identity", fill = "steelblue") +
  geom_line() +
  geom_point() +
  labs(
    title = "Scree plot",
    x = "Principal Components",
    y = "Proportion of Variance Explained"
  ) +
  pca_plot_theme

Key takeaways

Tip

  • PCA uses numeric data only
  • Missing values must be handled before PCA
  • Metadata must match the same samples used in PCA
  • Start simple, then build complexity
